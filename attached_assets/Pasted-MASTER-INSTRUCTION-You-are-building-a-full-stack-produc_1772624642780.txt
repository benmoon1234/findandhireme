SECTION 1 — PROJECT OVERVIEW & GOALS
Build a multi-role, multi-region job aggregation and recruitment platform that:

Allows job seekers to search, filter, save, and apply for jobs by keyword, skill, and location
Allows employers and organisations to register, post jobs, and manage applicants
Integrates third-party job aggregator APIs and RSS feeds with Pay-Per-Click (PPC) revenue tracking
Delivers AI-powered CV parsing and job-to-candidate matching with automated email/push alerts
Provides an admin panel for full platform management, moderation, and analytics
Covers dedicated job search experiences for the United States, United Kingdom, Nigeria, and a Global / Remote catch-all
Generates revenue through PPC aggregator partnerships, employer subscription tiers, and featured job listings


SECTION 2 — TECH STACK
Use the following technology stack. Do not deviate unless a listed tool is unavailable in the Replit environment, in which case use the closest equivalent and explain the substitution.
Frontend

Framework: Next.js 14 (App Router, React Server Components where appropriate)
Styling: Tailwind CSS with a custom design system (no generic purple-gradient AI aesthetics)
UI Components: shadcn/ui component library
Maps: Mapbox GL JS (for interactive map job search view)
Rich Text Editor: TipTap (for employer job description editing)
Icons: Lucide React
State Management: Zustand (global client state) + React Query / TanStack Query (server state, caching, API calls)
Forms: React Hook Form + Zod (validation)
Animations: Framer Motion (page transitions, micro-interactions)

Backend

Runtime: Node.js
Framework: Next.js API Routes (for simple endpoints) + a dedicated Express.js or Fastify server for heavier microservices (aggregator ingestion, CV parsing, notifications)
ORM: Prisma (with PostgreSQL)
Authentication: NextAuth.js (OAuth 2.0 — Google + LinkedIn providers) + JWT refresh token rotation
File Uploads: Multer (server-side) + AWS S3 SDK (storage) — or use Replit's object storage if S3 is unavailable
Email: Nodemailer with SendGrid transport
Push Notifications: Firebase Cloud Messaging (FCM)
RSS Feed Parsing: rss-parser npm package
Job Matching / CV Parsing: Use OpenAI API (GPT-4o) to extract structured data from uploaded CVs and perform semantic job-to-candidate matching scoring

Database

Primary: PostgreSQL (use Replit's built-in PostgreSQL or Neon serverless Postgres)
Search: Use PostgreSQL full-text search with tsvector and tsquery as the primary search engine. If Elasticsearch is available, use it instead.
Cache: Redis (use Upstash Redis for serverless compatibility with Replit)

Infrastructure & Services

Background Jobs: Use node-cron for scheduled tasks (aggregator feed polling, alert processing, backup triggers)
Rate Limiting: express-rate-limit + Redis store
Security: helmet.js, cors, express-validator, bcryptjs
API Key Management: Store all secrets in Replit Secrets (environment variables). Never hardcode secrets.
Payments: Stripe (for employer subscriptions and aggregator billing)

SECTION 8 — AUTHENTICATION SYSTEM
8.1 NextAuth Configuration (lib/auth.ts)

Configure Google OAuth provider
Configure LinkedIn OAuth provider
Configure Credentials provider (email + password with bcryptjs)
On first OAuth sign-in, auto-create a User and JobSeekerProfile record
JWT strategy with 15-minute access token expiry
Custom session callback that includes the user's role and id
Refresh token rotation: store refresh token in HTTP-only cookie, rotate on every access token refresh, invalidate entire session family if a refresh token is reused (detect theft)

8.2 Registration Page

Full name, email, password, confirm password fields
Role selection: "I'm looking for a job" or "I'm hiring"
Password strength indicator (zxcvbn library)
Email verification flow: send verification link via SendGrid on registration
On submit: hash password with bcryptjs (cost factor 12), create User + appropriate profile record
Check password against HaveIBeenPwned k-Anonymity API before saving; reject if found in breach database

8.3 Login Page

Email/password form + Google button + LinkedIn button
"Forgot Password?" link triggers reset email flow (time-limited token, 1-hour expiry)
After 5 failed attempts: lock account for 30 minutes, send alert email
Show human-readable error messages without exposing whether the email exists (prevent user enumeration)

8.4 Route Protection

All /dashboard/*, /employer/* routes: require authenticated session, redirect to /login if unauthenticated
All /admin/* routes: require role === SUPER_ADMIN
Employer-specific routes: require role === EMPLOYER
Use Next.js middleware (middleware.ts) for server-side route protection

CREATE INDEX idx_job_search ON "JobListing" USING GIN (search_vector);
CREATE OR REPLACE FUNCTION update_job_search_vector() RETURNS TRIGGER AS $$
BEGIN
  NEW.search_vector := to_tsvector('english', coalesce(NEW.title, '') || ' ' || coalesce(NEW.description, '') || ' ' || coalesce(NEW.location, ''));
